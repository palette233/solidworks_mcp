## 通过基准面获取对应物理面的可行性

### 💡 理论上可行，但有限制

#### 基准面与物理面的关系

**基准面（Datum Plane）**：
- 是一个**无限大的参考平面**
- 没有边界
- 不是实体几何

**物理面（Face）**：
- 是实体几何的一部分
- 有明确的边界
- 属于某个 Body

**关键问题**：基准面和物理面是**不同类型的实体**，没有直接的对应关系。

---

### ❌ 为什么直接对应不可行

1. **一个基准面可能对应多个物理面**- 例如：上视基准面可能与多个零件的顶面共面

2. **基准面是无限的，物理面是有限的**
   - 无法通过基准面唯一确定一个物理面

3. **SolidWorks API 没有提供直接映射**
   - 没有 `GetFacesOnPlane()` 这样的 API

---

### ✅ 可行的间接方案

#### 方案 A：通过基准面的法向量和位置筛选

**原理**：找到与基准面共面的物理面

**步骤**：
```
1. 获取基准面的法向量和位置（需要扩展 MCP）
2. list_entities(Face, componentName)
3. 对每个 Face，计算其法向量和中心点
4. 找到与基准面共面的 Face
5. select_entity(Face, matched_index, componentName)
```

**限制**：需要扩展 MCP 添加获取基准面属性的功能。

---

#### 方案 B：使用基准面创建草图，再投影到物理面

**原理**：利用 SolidWorks 的草图投影功能

**步骤**：
```
1. 在基准面上创建草图
2. 使用 SketchUseEdge 投影相邻的物理面边缘
3. 通过投影的边缘反推物理面
```

**限制**：复杂且不直接。

---

#### 方案 C：人工建立映射表（最实用）⭐

**原理**：手动记录基准面对应的物理面索引

**实现**：
```json
{
  "FL5A项目号700012.01-1": {
    "基准面4": {
      "对应物理面": "Face",
      "index": 5,
      "description": "顶部大平面，y=0.015"
    }
  }
}
```

**优点**：
- 简单可靠
- 一次配置，多次使用
- 不需要复杂的几何计算

---

### 🎯 推荐方案

**方案 C（映射表）+ 辅助工具**：

1. **首次配置**：在 UI 中选中物理面，运行辅助工具记录索引
2. **保存映射**：将"基准面名称 → 物理面索引"保存到配置文件
3. **后续使用**：直接从配置文件读取索引，使用 `select_entity`

这是当前最实用的方案，不需要扩展 MCP 的复杂功能。




## 为什么 append 功能无法解决组件内部基准面的选择问题

### 核心原因

**`append` 参数只解决"追加选择"的问题，不解决"能否选中"的问题。**

### 详细解释

#### append 功能的作用
```
select_by_name("A", "PLANE")// 选中 A，清除其他
select_by_name("B", "PLANE", append=true) // 保留 A，追加选中 B
```

**append 只是控制是否清除之前的选择，不改变选择的能力范围。**

---

#### 组件内部基准面无法选中的根本原因

这是 **SolidWorks API 的限制**，与 append 无关：

```csharp
// SelectByID2 的签名
bool SelectByID2(
    string name,        // 实体名称
    string type,        // 类型
    double x, y, z,     // 坐标
    bool append,        // 是否追加
    int mark,           // 标记
    object callout,     // 标注
    int selectOption    // 选项
)
```

**关键问题**：`name` 参数只能识别**当前文档顶层**的实体名称。

在装配体中：
- ✅ 可以选中：装配体自己的基准面（如"前视基准面"）
- ❌ 无法选中：组件内部的基准面（如"FL5A项目号700012.01-1/基准面4"）

---

### 类比说明

这就像文件系统：

```
装配体/
├── 前视基准面          ← select_by_name 能找到
├── bottomFace1         ← select_by_name 能找到
└── FL5A项目号700012.01-1/
    └── 基准面4         ← select_by_name 找不到（在子目录里）
```

`append` 只是让你能同时选中"前视基准面"和"bottomFace1"，但不能让你选中"基准面4"。

---

### 结论

添加 `append` 功能后：
- ✅ 可以同时选中多个**顶层**基准面
- ❌ 仍然无法选中**组件内部**的基准面

这是 SolidWorks API 的设计限制，不是 MCP 的实现问题。
